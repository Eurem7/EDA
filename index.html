1. Introduction

The Titanic dataset provides passenger information, including socio-economic status, ticket details, and survival status. This EDA aims to uncover patterns, handle missing values, detect anomalies, and extract meaningful features for further modeling.

2. Data Overview

The dataset consists of 418 records and 12 columns, spanning categorical (e.g., Sex, Embarked),numerical (e.g., Age, Fare), and mixed-type features (Cabin, Ticket). Key observations include:

‚Ä¢ Missing values: Age (~20%), Fare (1 entry), and Cabin (77%) have null values.

‚Ä¢ Data types: Some categorical variables require encoding for machine learning compatibility.

‚Ä¢ Potential feature interactions: Family relationships (SibSp, Parch) and passenger class (Pclass) likely impact survival rates.

3. Data Cleaning & Preprocessing

To ensure robust analysis, the following steps were taken:

~ Missing Values:

‚Ä¢ Age was imputed using the median (resistant to outliers).

‚Ä¢ Fare was imputed using the median to maintain data consistency.

‚Ä¢ Cabin was dropped due to excessive missing data (~77%).

~Categorical Encoding:

‚Ä¢ Sex and Embarked were transformed using Label Encoding.

‚Ä¢ Embarked missing values were replaced with the most frequent category (mode).

~ Feature Engineering:

‚Ä¢ Title Extraction: Titles (Mr., Mrs., Miss., etc.) were extracted from names to explore their impact on survival.

‚Ä¢ Family Size: A new feature FamilySize = SibSp + Parch + 1 was created to analyze family influence.
